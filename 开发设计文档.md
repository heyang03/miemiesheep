# Shopify Product Collector 插件开发设计文档

版本：V1.0
项目类型：Chrome Extension Manifest V3
项目目标：采集当前页面商品信息，并导出为 Shopify 可导入 CSV
当前状态：需求与技术方案设计完成，待进入代码开发阶段

---

## 1. 项目背景

本项目需要开发一个谷歌浏览器插件，用于在用户浏览商品详情页时，自动采集当前页面的商品信息，并导出为 Shopify 可导入的 CSV 文件。

插件第一版以“稳定可用”为目标，不追求一开始适配所有网站，而是先完成：

打开商品页 → 点击插件 → 采集商品信息 → 弹窗预览和修正 → 导出 Shopify CSV。

---

## 2. 项目目标

### 2.1 第一阶段目标

第一阶段开发一个不依赖后端的 Chrome 插件，支持：

1. 采集当前商品详情页信息。
2. 识别商品标题、价格、描述、图片、SKU、品牌等基础数据。
3. 在插件弹窗中展示采集结果。
4. 支持用户手动修改采集结果。
5. 导出 Shopify 可导入 CSV。
6. 支持中文内容，CSV 使用 UTF-8 with BOM。
7. 简单商品默认生成一个 `Default Title` 变体。
8. 多图片按照 Shopify CSV 图片行规则导出。

### 2.2 第一版不做的内容

第一版暂不做：

1. 不做 Shopify Admin API 直接上架。
2. 不做 Shopify OAuth 授权。
3. 不做后端接口。
4. 不做数据库。
5. 不做批量列表页采集。
6. 不做自动登录采集。
7. 不做图片下载和重新托管。
8. 不做 AI 标题优化和描述改写。
9. 不做复杂变体自动识别。
10. 不做绕过反爬、付费墙或权限限制。

## 6. 插件工作流程

```txt
用户打开商品详情页
    ↓
点击 Chrome 插件图标
    ↓
打开 popup.html
    ↓
点击“采集当前商品”
    ↓
popup.js 向当前页面发送消息
    ↓
content.js 读取当前页面 DOM、JSON-LD、Meta 信息
    ↓
返回标准商品对象 ProductDraft
    ↓
popup.js 将数据渲染到表单
    ↓
用户检查并手动修改
    ↓
点击“导出 Shopify CSV”
    ↓
csv.js 生成 CSV
    ↓
浏览器下载 CSV 文件
```

## 7. Chrome 插件权限设计

### 7.1 第一版权限

```json
{
  "permissions": ["activeTab", "scripting"],
  "host_permissions": ["<all_urls>"]
}
```

### 7.2 权限说明

| 权限               | 作用                 |
| ---------------- | ------------------ |
| activeTab        | 用户点击插件时，允许访问当前活动页面 |
| scripting        | 允许插件与页面脚本交互        |
| host_permissions | 允许插件在指定网站范围运行      |

### 7.3 后续优化方向

第一版为了开发方便，可以使用 `<all_urls>`。

后续正式发布前建议优化：

1. 尽量依赖 `activeTab`。
2. 减少长期全站权限。
3. 支持用户手动添加允许采集的网站。
4. 插件说明中明确不上传用户数据。

## 8. 商品采集策略

### 8.1 采集优先级

```txt
JSON-LD schema.org/Product
    ↓
Open Graph / Meta 信息
    ↓
常见 DOM 选择器
    ↓
图片兜底
    ↓
用户手动修正
```

---

## 9. JSON-LD 采集规则

优先读取页面中的：

```html
<script type="application/ld+json">
```

识别：

```txt
@type = Product
```

支持结构：

```txt
单个 Product
Product 数组
@graph 中的 Product
```

可采集字段：

| JSON-LD 字段           | 插件内部字段       |
| -------------------- | ------------ |
| name                 | title        |
| description          | description  |
| image                | images       |
| sku                  | sku          |
| brand.name           | vendor       |
| offers.price         | price        |
| offers.priceCurrency | currency     |
| offers.availability  | availability |

---

## 10. Meta 信息采集规则

兜底读取：

| Meta 字段                  | 插件内部字段      |
| ------------------------ | ----------- |
| og:title                 | title       |
| og:description           | description |
| og:image                 | images      |
| meta[name="description"] | description |
| twitter:title            | title       |
| twitter:image            | images      |

---

## 11. DOM 兜底采集规则

当 JSON-LD 和 Meta 数据不足时，从页面 DOM 中采集。

| 内容  | 选择器策略                               |
| --- | ----------------------------------- |
| 标题  | h1、class 包含 title、product-title 的元素 |
| 价格  | class/id 包含 price 的元素、data-price    |
| 描述  | class/id 包含 description 的元素         |
| 图片  | 页面 img、picture source、商品图区域图片       |
| SKU | 文本包含 SKU、Item No、Product Code 的元素   |

---

## 12. 图片采集规则

### 12.1 图片来源优先级

```txt
JSON-LD image
    ↓
og:image
    ↓
页面商品图区域
    ↓
页面 img 标签兜底
```

### 12.2 图片处理规则

1. 相对路径转绝对 URL。
2. 图片 URL 去重。
3. 过滤空 URL。
4. 过滤明显的 icon、logo、svg。
5. 默认最多保留 10 张图片。
6. 用户可以在弹窗中删除不需要的图片。
7. 第一版不下载图片。
8. 第一版不重新托管图片。

### 12.3 Shopify 图片要求

导出的 CSV 里只能写图片 URL。

图片 URL 必须：

1. 公网可访问。
2. 推荐 `https://`。
3. 不需要登录。
4. 不返回 403。
5. 不是本地路径。
6. 不是浏览器 blob 地址。
7. 不是 base64。

---

## 13. 价格采集规则

价格需要标准化为数字字符串。

示例：

```txt
"$19.99"      → "19.99"
"USD 19.99"   → "19.99"
"￥129.00"    → "129.00"
"1,299.00"   → "1299.00"
```

第一版不做汇率转换。

## 18. 多图片 CSV 规则

### 18.1 第一张图片

第一张图片写在商品主行。

```txt
Title = 商品标题
URL handle = 商品 handle
Product image URL = 第一张图片 URL
Image position = 1
Image alt text = 商品标题
```

### 18.2 第二张及后续图片

后续图片新增独立图片行。

只填写：

```txt
URL handle
Product image URL
Image position
Image alt text
```

其他字段留空。

### 21.2 风险提示

| 情况         | 处理                |
| ---------- | ----------------- |
| 价格为空       | 提示用户，将默认使用 0.00   |

| SKU 为空     | 允许导出              |

## 22. CSV 转义规则

CSV 需要处理：

1. 英文逗号。
2. 双引号。
3. 换行。
4. 中文。
5. HTML 描述。

规则：

```txt
包含逗号、双引号、换行的字段，用双引号包裹
字段内部双引号替换为两个双引号
文件编码使用 UTF-8 with BOM
```

示例：

```txt
原始值：
Women's "Classic", T-Shirt

CSV 值：
"Women's ""Classic"", T-Shirt"
```

## 24. 第一版验收标准

插件必须满足：

1. Chrome 能正常加载插件。
2. 点击插件图标能打开弹窗。
3. 点击采集按钮能读取当前页面信息。
4. 能采集标题。
5. 能采集价格或允许手动填写。
6. 能采集描述或允许手动填写。
7. 能采集图片或允许无图导出。
8. 能手动修改字段。
9. 能导出 CSV 文件。
10. CSV 第一行字段正确。
11. CSV 中文不乱码。
12. CSV 描述中有逗号、引号、换行时不乱列。
13. 多图片能生成多行。
14. Shopify 后台可以识别 CSV。
15. 简单商品可以导入为草稿产品。
