# P4 性能与体验优化任务清单

目标：在不改变现有采集、导出和消息语义的前提下，降低插件对页面和侧边栏的性能开销，减少后续维护成本，并继续提升产品化体验。

## 优化任务

1. [已完成] P0 - 按需注入 `content.js`
   - 现状：`manifest.json` 通过 `<all_urls>` 常驻注入 `content.js`。
   - 目标：移除常驻 content script，复用 `sendMessageWithInjection` 在采集、选图、站点规则等操作发生时按需注入。
   - 验收：采集当前商品、图片选择、站点规则、批量发现相关测试仍可用；普通网页不再默认加载采集脚本。

2. [已完成] P0 - 图片/变体列表事件委托与分片渲染
   - 现状：`renderImages`、`renderVariants` 每次重建全部 DOM，并给每个控件独立绑定事件。
   - 目标：容器级事件委托，批量 DOM 用 `DocumentFragment`，大列表时分片渲染或分页。

3. [已完成] P0 - 图片检查并发限制与结果缓存
   - 现状：图片检查对所有图片 `Promise.all` 并发请求。
   - 目标：限制并发到 4-6 个，同 URL 复用检查结果，降低网络和 UI 峰值压力。

4. [已完成] P1 - 表单保存路径增量化
   - 现状：输入保存时经常全量读取商品、图片、变体 DOM。
   - 目标：普通字段直接写 draft，图片/变体只更新当前 index，必要时再做全量 normalize。

5. [待处理] P1 - 采集逻辑分阶段短路
   - 现状：单次采集会跑多种来源和 DOM 兜底扫描。
   - 目标：高置信来源先返回核心字段，缺失字段再 fallback；重型 DOM/HTML 扫描延后。

6. [待处理] P1 - CSS 清层与设计 token 收敛
   - 现状：旧绿色体系与 P3 新样式并存，靠后续规则覆盖。
   - 目标：整理为 token/base/components/popup 结构，移除旧色板和重复覆盖规则。

7. [待处理] P2 - 滚动结构收敛
   - 现状：主容器、图片列表、变体列表和底部栏存在多层滚动/固定关系。
   - 目标：侧边栏优先单主滚动；窄屏下图片和变体区域自然流展示，减少错位和遮挡。

8. [待处理] P2 - 测试工程统一
   - 现状：测试脚本的 Playwright 启动方式、Chrome mock 和 storage mock 不统一。
   - 目标：抽取 `tools/test-utils.mjs`，统一浏览器 fallback、mock 注入和常用断言。
